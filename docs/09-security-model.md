# Security Model

## Purpose
Define baseline security and governance model for CT.

## Scope
Secrets, RBAC, audit logs, privacy, AI security, and CI security checks.

## Current Status
⚫ Baseline controls documented.

## Key Components
- Secrets policy
- Role-based access
- Audit logging
- Document privacy controls
- RAG prompt-injection risk controls
- MCP tool permissions
- Dependency scanning
- Branch protection recommendations
- GitHub Advanced Security recommendations
- Private repo assumptions
- No PII in logs
- No keys in commits

## Risks
Privilege escalation and sensitive data leakage.
