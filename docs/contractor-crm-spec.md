# Contractor CRM Spec

## Purpose
Model and control contractor lifecycle from identification through engagement.

## Scope
Companies, contacts, trades, qualifications, insurance/licenses, communication, follow-up, quote and contract status.

## Current Status
🟡 Domain types and guardrails are implemented.

## Key Components
- ContractorCompany
- ContractorContact
- TradeCategory
- SubcontractorProfile
- VendorQualification
- LicenseCredential
- InsuranceRecord
- W9TaxRecord
- SafetyRecord
- PriorProjectReference
- QuoteBid
- ScopePackage
- EngagementContract
- ComplianceStatus
- CommunicationLog
- FollowUpTask

## Risks
Unverified credentials or engagement without approvals.
