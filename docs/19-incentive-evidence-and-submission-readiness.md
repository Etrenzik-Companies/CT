# Incentive Evidence and Submission Readiness

## Incentive Evidence Stages

- `estimated`
- `application_ready`
- `submitted`
- `awarded`
- `verified`
- `rejected`
- `expired`

## Required Controls

- Incentive values must be separated as estimated, submitted, awarded, and verified.
- Estimated incentives must never count as verified funds.
- Uploaded application material is not the same as an accepted award.
- Verification requires reviewed award evidence plus accounting review.

## Submission Readiness Rules

- Lender-ready status requires reviewed evidence, not just uploaded documents.
- RAG outputs must cite indexed source documents.
- Blockchain and XRPL references are proof references only unless separately approved.
- Legal, accounting, lender, and human review are required before any real-world submission.
