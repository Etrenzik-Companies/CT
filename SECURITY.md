# Security Policy

## Security Defaults

- No secrets in source control.
- No real API keys in repository files.
- Use role-based access controls for mutating operations.
- Keep audit logs for sensitive actions.
- Treat all AI outputs as assistive, not authoritative.

## Reporting a Vulnerability

Report privately to security@placeholder.invalid with reproduction steps and impact.

## Recommendations

- Enable branch protection on `main`.
- Enable GitHub Advanced Security (when available).
- Enable dependency and secret scanning.
- Require PR reviews for privileged code areas.
